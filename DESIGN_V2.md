## D4 🔬結晶の目（buildMicroScene・板金オリジナル）

曲げ断面の原子格子。曲げが進むと**外側=格子間隔が開く（引張・青→赤に色変化）／内側=詰まる（圧縮）／中央=中立軸（不変・緑）**。モード: 曲げ系レッスン=上記基本／スプリングバック系=荷重を離すと弾性分だけ格子が戻る動き／割れ系=外側の結合が限界で切れる（R小さすぎの正体）。対応表を報告書に記載。チップ「※分子イメージ図」。

## D6 判断ドリル24問（別紙・全文このまま実装）

tag: 曲げR(radius)/展開(unfold)/穴と縁(hole)/公差(tol)/材料(matl)/コスト(cost)
1. radius1: 内R=0（ピン角）を図面指定した → **NG**「外側の材料が伸びきれず割れる。金型も痛む」→「対策: 内R≧板厚が目安。SUSやアルミ硬材はもっと大きく」
2. radius2: 曲げ線のすぐ近くに穴を配置した → **NG**「曲げで穴が引っ張られ楕円に変形する」→「対策: 穴フチ〜曲げまで約2×板厚＋R以上。無理なら曲げ後加工」
3. radius3: SPCC（冷間圧延鋼板）を内R＝板厚で曲げた → **OK**「軟鋼の標準的な安全ライン」→「バネ材・高張力材は別基準」
4. radius4: バネ材を金型なりの直角ピッタリで狙った → **NG**「スプリングバック（弾性の戻り）で開く」→「対策: オーバーベンド（曲げすぎて戻す）か現物合わせ前提の指示」
5. unfold1: 展開長を外寸の単純足し算で出した → **NG**「曲げ部は伸びる。足し算では長すぎる」→「対策: 曲げ代（ボンドデダクション）を引く。中立軸のズレが正体」
6. unfold2: CADの板金機能で展開し、K係数（中立軸位置）を現場実績に合わせた → **OK**「展開精度は係数合わせが9割」→「加工先とV幅・係数を共有すると最強」
7. unfold3: 4辺全部を立ち上げる箱を、曲げ順を考えず設計した → **NG**「最後の曲げで工具と壁が干渉して曲がらない」→「対策: 頭の中でプレスブレーキに載せて順番に曲げてみる」
8. unfold4: 曲げ線を圧延目（ロール目）と直角になるよう配置した → **OK**「目に平行な曲げは割れやすい」→「板取り（ネスティング）と両立させる」
9. hole1: 曲げ交差部に逃がしの切欠きを入れた → **OK**「角の破れ・しわの防止定石」→「小さな逃げが仕上がりを大きく変える」
10. hole2: タップ穴を曲げ線のすぐ横に置いた → **NG**「曲げの歪みでタップが斜め・不良になる」→「対策: 距離を取るか曲げ後にタップ」
11. hole3: 板厚1.6に対しφ1.0の丸穴をパンチ抜きで指定した → **NG**「板厚より小さいパンチは折れる」→「対策: 穴径≧板厚が目安。レーザーなら可の場合も（加工先に確認）」
12. hole4: 薄板のねじ止めにバーリング＋タップを使った → **OK**「立ち上げで山数を稼ぐ板金の知恵」→「M3〜M5でよく使う。下穴径が命」
13. tol1: 板金の全寸法に±0.05を指定した → **NG**「板金の現実は曲げ±0.1〜0.3。全部厳しいと全部高い」→「対策: 効く寸法だけ絞り、他は一般公差」
14. tol2: 基準面・基準穴を決めてそこから寸法を追った → **OK**「基準が揺れると全部揺れる」→「加工も検査も基準で楽になる」
15. tol3: 溶接をまたいだ精密寸法を指定した → **NG**「溶接は熱で歪む。狙えない」→「対策: 溶接後加工にするか、公差を歪み込みで設定」
16. tol4: 「どの寸法が機能に効くか」を加工先と共有した → **OK**「現場は優先順位が分かると強い」→「図面は手紙。読む人を想像」
17. matl1: 亜鉛メッキ鋼板の切断端面の防錆を検討した → **NG判定に注意→OK**「端面は亜鉛がなく裸の鉄。検討は正しい」→「犠牲防食である程度守られるが、環境次第で塗装・別材を選ぶ」
18. matl2: SUSをSPCCと同じ曲げ条件でプログラムした → **NG**「SUSはスプリングバック大・必要トン数大」→「対策: 材料ごとの条件表。SUSは戻り込みで曲げる」
19. matl3: 板厚を流通標準（1.0/1.2/1.6/2.3/3.2）から選んだ → **OK**「特殊板厚は納期と価格が跳ねる」→「設計の自由は流通の中で使う」
20. matl4: ヘアライン材（意匠面）を保護フィルム指定なしで手配した → **NG**「加工中の傷＝即不良」→「対策: フィルム指定＋フィルム剥がしの工程も考える」
21. cost1: 複雑な多曲げ1品物を、曲げ数を減らして溶接組立に割った → **OK**「曲げ段取りも工数。分けた方が安いことがある」→「数量で最適解が変わる（量産は逆もある）」
22. cost2: 標準金型で曲がらない特殊角度・特殊Rを多用した → **NG**「特殊金型＝お金と納期」→「対策: 加工先の金型リストの範囲で設計する」
23. cost3: 展開形状の板取り（ネスティング）を意識して歩留まりを上げた → **OK**「材料費は形で決まる」→「定尺板からの取り数を想像する」
24. cost4: 点付けで十分な内部部品に全周溶接を指定した → **NG**「歪む・時間かかる・研磨も増える」→「対策: 必要な強度・水密だけ全周。他は点付け・断続」
